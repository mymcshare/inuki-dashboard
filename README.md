# 居抜き物件ダッシュボード 🍜

ラーメン出店候補の居抜き物件（1都3県）を毎日自動収集・表示するダッシュボードです。

## 公開URL

**https://inuki-dashboard.ict-share.workers.dev**

---

## 使い方

### 基本的な見方

| バッジ | 意味 |
|--------|------|
| ◎ 有力 | 重飲食可 ＋ 立地・厨房・規模が好条件 |
| ○ 候補 | 重飲食可で標準的な条件 |
| △ 要検討 | 軽飲食のみ等、改装が必要 |

**NEW** タグが付いている物件は最新3日以内の新着です。

### フィルター機能

- **都県でしぼる** — 東京・神奈川・千葉・埼玉を個別に切り替え
- **賃料の上限** — 上限を指定して絞り込み
- **面積（坪）** — 下限・上限を指定
- **駅から徒歩** — 3分・5分・10分以内で絞り込み
- **キーワード** — 地名・駅名・前業態でフリーワード検索
- **重飲食可のみ** — デフォルトON（ラーメン出店可能物件のみ表示）
- **厨房を流用しやすい前業態のみ** — ラーメン・中華・居酒屋など厨房設備が活かせる物件
- **1階・路面のみ** — 路面店のみに絞り込み

### 並び替え

テーブルの列見出し（最寄り駅・賃料・面積・登録日）をクリックして並び替えできます。再クリックで昇順⇄降順が切り替わります。

---

## データについて

- **対象エリア** — 東京都・神奈川県・千葉県・埼玉県
- **対象物件** — 居抜き物件（飲食店ドットコム掲載）
- **自動更新** — 毎日 23:50 JST に最新データを取得
- **保持件数** — 最大200件（古いものから順に削除）

## システム構成（管理者向け）

```
GitHub (mymcshare/inuki-dashboard)
  ├── index.html          ダッシュボード本体
  ├── scraper/scrape.py   スクレイパー（Python）
  └── .github/workflows/
        daily-update.yml  毎日23:50 JST に自動実行
              ↓
        index.html の DATA配列・更新日時を書き換えてcommit & push
              ↓
Cloudflare Pages が自動再デプロイ → URL に即時反映
```

### ローカルでスクレイパーを手動実行する場合

```bash
cd inuki-dashboard
pip install -r scraper/requirements.txt
python scraper/scrape.py
```

実行後、`index.html` が更新されます。GitHubにpushするとダッシュボードに反映されます。
